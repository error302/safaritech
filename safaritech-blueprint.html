// M-Pesa STK Push — tRPC mutation example
export const paymentRouter = createTRPCRouter({
  initiateMpesa: protectedProcedure
    .input(z.object({ orderId: z.string(), phone: z.string() }))
    .mutation(async ({ ctx, input }) => {
      const order = await ctx.db.order.findUniqueOrThrow({ where: { id: input.orderId } });
      const token = await getMpesaToken();
      const res = await stkPush({ token, phone: input.phone, amount: order.total });
      return { checkoutRequestId: res.CheckoutRequestID };
    }),
});

09 — Deployment

Google Cloud VM Deployment

1
Provision VM
e2-medium (2 vCPU, 4GB RAM) in us-central1 or africa-south1. Enable HTTP/HTTPS firewall. Attach static IP.
2
Install Docker + NGINX
Install Docker CE, docker-compose, certbot. Configure NGINX as reverse proxy pointing to container port 3000.
3
SSL Certificate
certbot --nginx -d voltique.co.ke -d www.voltique.co.ke — auto-renews via cron.
4
Deploy App
docker-compose up -d --build — builds and starts Next.js app container in detached mode.
5
Run DB Migrations
npx prisma migrate deploy — safe to run on every deploy, only applies pending migrations.
6
Cloudflare DNS
Point A record to VM IP. Enable Cloudflare proxy (orange cloud). Set SSL to "Full (strict)". Enable auto-minify.
7
GitHub Actions CI/CD
On push to main: typecheck → lint → test → build → SSH to VM → docker pull + restart. Zero-downtime via rolling restart.

10 — Growth

Scaling Strategy

Phase 1 Single VM 0–10K users/month. One GCP VM, Neon Postgres free tier, Upstash Redis free tier. Cost ~$10–20/mo.
Phase 2 VM Upgrade + CDN 10K–100K users. Upgrade VM to e2-standard-4. Neon paid, Redis paid. Cloudflare caching aggressive. Cost ~$50–100/mo.
Phase 3 Load Balancer + Multiple VMs 100K–500K users. GCP Load Balancer + 2-3 VMs. Cloud SQL PostgreSQL. Cloud Memorystore Redis. Cost ~$200–500/mo.
Phase 4 GKE + Microservices 500K+ users. Google Kubernetes Engine. Separate services: auth, products, orders, payments. Autoscaling, zero downtime. Cost scales with usage.

12 — Principles

Golden Rules

01
Type EverythingIf TypeScript complains, fix it. Never @ts-ignore. Types are documentation that can't lie.
02
Validate at the BoundaryZod schemas on every tRPC input. No raw user data ever touches the database.
03
Verify WebhooksAlways verify Stripe and M-Pesa webhook signatures. Never trust a callback without checking the signature.
04
No Blank ScreensEvery route needs loading.tsx and error.tsx. Users should never stare at white.
05
Secrets Are Sacred.env.local is gitignored. Doppler syncs to production. One leaked key = full breach.
06
Test Payments in SandboxRun full checkout flows in Stripe test mode AND M-Pesa sandbox before touching production keys.
07
Deploy EarlyPush a landing page on Day 1. Real feedback from real users beats planning in isolation every time.
08
Optimistic UI AlwaysCart add, wishlist toggle — update the UI instantly, sync to server in background. Never make users wait.
